直接在对话中提出以下任一问题，skill 应该会自动触发。

## ✅ 验证清单

对于每个测试用例，检查以下内容：

- [ ] **需求分析** - 是否正确理解了用户需求
- [ ] **组件推荐** - 推荐的组件是否合适（不超过 3 个主要组件）
- [ ] **安装命令** - 是否提供了安装命令（在任何代码示例之前）
- [ ] **代码示例** - 是否提供了完整的、可运行的代码示例
- [ ] **TypeScript** - 代码示例是否使用 TypeScript
- [ ] **最佳实践** - 是否提供了使用建议
- [ ] **文档链接** - 是否提供了相关文档链接（如有）
- [ ] **格式规范** - 输出格式是否符合要求（推荐组件 → 安装命令 → 代码示例 → 最佳实践）

## 🐛 常见问题

### 问题 1: Skill 没有触发

**解决方案**:
- 确保使用 `/kesi-ui` 命令
- 或确保问题包含关键词："组件"、"表单"、"表格"、"地图"、"图表"

### 问题 2: 没有提供安装命令

**解决方案**:
- 检查 skill 的 frontmatter 和响应指令部分
- 确保指令明确要求先提供安装命令

### 问题 3: 代码示例不完整

**解决方案**:
- 检查对话示例部分是否提供了完整的示例
- 确保示例包含 import 语句和完整的组件使用

### 问题 4: 组件数量不匹配

**解决方案**:
- 运行 `find .claude/skills/kesi-ui/data -name "*.md" | wc -l` 确认实际文件数
- 更新 SKILL.md 中的统计信息
